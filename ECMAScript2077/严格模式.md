# 严格模式

在JavaScript发展的过程中，一直遵循向下兼容的策略，新的功能被加入，旧的功能继续存在。这样做的目的是为了兼容老旧项目，让已有代码继续正常运行，但这也意味着JavaScript中的一些设计上的糟粕也一直存在。

例如在非严格模式下，下面这段代码并不会报错，而是会创建新的变量：

```javascript
a = 1;
```

在某些情况下，这可能并不符合预期。在「严格模式」下，上面这段代码将会抛出异常。

**通过启用「严格模式」，可以让代码的执行更符合预期，从而减少错误发生的几率。**

**「严格模式」的出现既可以让我们屏蔽掉JavaScript中糟粕的部分，同时还可以允许旧的代码正常工作。**
